## Análisis realizado

Durante la práctica se analizaron diferentes aspectos del tráfico de red:

### 1. Captura de tráfico

Se realizó una captura de paquetes utilizando Wireshark mientras se generaba tráfico mediante navegación web.

Se observaron protocolos como:

- TCP
- UDP
- DNS
- HTTP/HTTPS
- TLS

---

### 2. Análisis DNS

Se estudiaron las consultas DNS realizadas por el equipo para resolver nombres de dominio en direcciones IP.

Aspectos observados:

- Dominio solicitado.
- Servidor DNS utilizado.
- Dirección IP obtenida.

---

### 3. Análisis de conexión web

Se analizó el proceso de conexión entre el cliente y un servidor web:

- Resolución DNS del dominio.
- Establecimiento de conexión TCP.
- Negociación TLS para conexiones HTTPS.
- Intercambio de paquetes.

---

### 4. Protocolos observados

| Protocolo | Función |
|-----------|---------|
| TCP | Transporte fiable de datos |
| UDP | Comunicación rápida sin conexión |
| DNS | Resolución de nombres de dominio |
| HTTP | Transferencia de páginas web |
| HTTPS/TLS | Comunicación cifrada |

---

## Conclusiones

El análisis con Wireshark permite visualizar cómo los dispositivos se comunican dentro de una red y entender el funcionamiento de los protocolos utilizados diariamente.

Esta práctica ha servido para familiarizarse con la captura de paquetes, la interpretación de tráfico y la identificación de posibles problemas o comportamientos anómalos en una red.
